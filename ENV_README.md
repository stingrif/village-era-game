# Переменные окружения

Единственный полный список ключей и комментариев: **[бэкенд/env.example](бэкенд/env.example)**. Здесь — только как задать `.env`.

---

## Бэкенд (папка `бэкенд`)

1. Скопируйте шаблон:
   ```bash
   cd бэкенд
   cp env.example .env
   ```
2. Откройте `.env` и подставьте значения (все ключи и пояснения — в [бэкенд/env.example](бэкенд/env.example)).
3. Обязательно для запуска: **DATABASE_URL**, при использовании кэша — **REDIS_URL**. Остальное по необходимости (см. env.example).

---

## Фронт (корень проекта Игра)

1. В корне проекта: `cp env.example .env` (если есть свой env.example для фронта).
2. В коде или в сгенерированном конфиге задайте **API_BASE** — URL бэкенда (например `http://localhost:8000`). См. также [бэкенд/RUN_LOCAL.md](бэкенд/RUN_LOCAL.md).

Детали переменных не дублировать: только в **бэкенд/env.example**.
