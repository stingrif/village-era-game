# Антибот и санкции

## Продажа токена

При детекте продажи токена проекта:

1. Аннулирование: era_points, withdraw_eligibility_value_30d, ranking_contribution  
2. Штраф **40%** от coins_balance в пользу проекта  
3. Уведомление, **3 часа** на пополнение (required_tokens из ENV)  
4. **token_compliance:** status, deadline_at, snapshot_json (coins_balance, era_points, и т.д.)  
5. Если дедлайн прошёл — restricted, вывод запрещён на срок  

## risk_profile (таблица)

- risk_score (0..1000), vpn_hits, multi_account_hits, status (ok|limited|blocked)

## Risk score — начисление

| Сигнал | + |
|--------|---|
| 3+ аккаунта один device_hash | 250 |
| 3+ аккаунта один wallet_hash | 180 |
| Чек-ин «идеально по таймеру» 10 раз | 120 |
| 50+ кликов шахты < 60 сек | 90 |
| Частая смена ip_hash | 80 |
| Обход лимитов | 60 |

| Снижение | − |
|----------|---|
| Подпись кошельком + стабильность 7 дней | 120 |
| Нормальная активность 7 дней | 60 |
| Реклама «человеческая» | 40 |

## Пороги

| Risk | Статус | Ограничения |
|------|--------|-------------|
| < 250 | normal | — |
| 250–449 | watch | +2 мин CD чекина, −1 попытка |
| 450–699 | restricted | вывод запрещён, маркет только покупка |
| ≥ 700 | banned | заморозка операций |

## user_flags (опционально)

- is_trade_allowed, is_withdraw_allowed, is_ads_allowed  
- risk_score, risk_label (normal|watch|restricted|banned)  

## user_identities

Хранятся хэши: device_hash, ip_hash, wallet_hash (не raw). Для авто-связки мультиакков и расчёта risk.
